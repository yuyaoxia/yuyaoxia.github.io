solr_two

发表于 2017-08-16   |   分类于 sorl

solr支持在单个引擎上运行多个内核，每个内核可视为单独的索引和配置，这样可以在一个服务器上管理多个内核，但每个服务器有且仅有一个包含所有内核的主目录。
solr.solr.home全局属性设置了solr主目录位置
linux上solr主目录在”/usr/solr-5.5.3/example/example-DIH/solr/solr/conf”下，
主要配置文件：solrconfig.xml、schema.xml

###基本查询属性说明

表单字段 取值 说明
q ipod 主查询参数；根据该参数中的词项与文档相似度，对文档评分 ；例如，q=id:1，默认为q=:
fq manu:Belkin 过滤查询；通过过滤器筛选结果集文档，但不影响评分。
sort price asc 指定排序字段和排序方式
start 0 指定搜索结果的起始页
rows 10 页面大小
fl name,price,score 在结果集中每个文档返回的字段列表。score是内置字段，用于保存每个文档查询的相关度得分。默认不返回
df text 没指定搜索字段时的默认搜索字段；text是示例服务器的全包含字段
wt xml 响应输出类型,有 xml, json, php等

###solr是一个文档存储与检索引擎，solr存储与更新删除的原理
solr处理文档是基于http协议的，所有的操作基层都是由HttpClient【支持HTTP协议的客户端编程工具包】完成
1、新增

2、修改，给需要修改的属性值加上set标识，然后做HttpSolrClient.add(Object obj)操作；solr有自己的协议，当看到有set标识时，在add操作时就做修改动作。如果没有set标识，solr在添加之前根据主键删除该条数据，然后把包括item_code、key仅这两字段的对象加到solr中

3、删除
