solr实战第一章--初识solr

发表于 2017-08-15   |   分类于 sorl

sorl是可扩展的、开箱即用的企业级搜索引擎，简单的说，就是一个文档存储与检索引擎。是一个JAVA WEB应用，是一种具体的NoSQL技术[非关系型数据存储和处理技术],用来搜索大规模文本数据并根据相关度排序结果，可以运行在任何主流Java servlet 引擎中，例如jetty和tomcat，或者JBoss、OracleAS这样的J2EE应用服务器。

solr不是什么

  • 不是Google或Bing这样的网络搜索引擎
  • solr不具备网站搜索引擎优化（SEO）方面的功能

    主要特征

  • 可扩展：通过集群中多台服务器的分布式运行实现扩展
  • 开箱即用：solr是开源的，易于安装和配置，并提供预先配置好的示例服务器，易上手
  • 为搜索优化：速度快，以亚秒级速度执行复杂查询，往往只需几十毫秒
  • 大规模文档：可以用于处理包含百万级文档的索引
  • 以文本为中心：针对自然文本搜索进行优化
  • 根据相关度对结果排序：根据文档与用户查询的相关程度对文档进行排序

    适于处理的数据表现

  • 以文本为中心
  • 读主导
  • 面向文档
  • 灵活的模式
  • 大量的数据需要处理（即大数据）

    提供了哪些内置分析工具【子系统】

  • 拼写检查[用户输入时提供建议]
  • 同义词处理
  • 短语查询
  • 文本分析

    信息检索引擎

    信息检索（information Retrieval,IR）是从大规模集合中查找满足特定信息要求的具有非结构化性质（通常是文本）的资料（通常是文档）过程。

    Solr是构建在apache的Lucene之上【Lucene是一个广受欢迎的基于java的开源信息检索库】；在后台，Lucene是一个Java类库,用于倒排索引的构建与管理【倒排索引是专门用于匹配查询此项与文档的数据结构】；apache的Hadoop提供的MapReduce开源实现，为Solr的web大规模搜索，构建lucene倒排索引
    Lucene查询与数据库查询的不同：lucene的结构是根据查询的相关度进行排名，而数据库只是根据某一列或者几列排序
    Lucene为solr提供文档索引和查询执行的类库，但缺少生产索引结构的简单配置方法；solr的schema.xml配置文件提供了一种简明的索引结构定义和字段表示及分析方法，在后台，solr使用该配置文件表示所有可能的字段和数据类型，将文档映射为Lucene的所有

    为什么选择sorl

  • 稳定性：sorl是个成熟的技术，可从开源代码的主工作目录直接部署
  • 可扩展性：solr的子系统是由模块化的“管道构成”，通过插件方式实现新功能，这样在已有管道中接入新的搜索构建就能扩展
  • 可伸缩性：solr自带缓存和增加服务器可以优美实现查询吞吐量的伸缩，通过索引分片分布式搜索优美实现文档索引量的伸缩
  • 容错性：solr会对索引分片添加副本，通过集群故障转移来避免故障

    sorl主要功能

    用户体验功能

  • 分页和排序
  • 分面：对搜索结果进行归类分组
  • 自动建议：减少错误查询的数量，提高精准搜索度
  • 拼写检查：支持自动更新（搜索拼写错误，solr自动更正）和你要找的是不是（solr建议查询可能得到更精准的搜索结果）
  • 搜索结果高亮：搜索量大时方便一眼看到搜索信息
  • 地理空间搜索：内置经纬度索引支持，根据位置距离排序

    数据建模功能

  • 结果分组/字段折叠：根据共性将结果分组也成字段折叠
  • 灵活的查询支持：
    1、“与”（AND）、或（OR）、非（NOT）条件逻辑
    2、通配符匹配
    3、支持日期和数字的区间查询
    4、支持词项间隔距离的短语查询
    5、模糊字符串匹配
    6、正则表达式匹配
    7、函数查询
  • 链接：join相当于sql的子查询
  • 文档聚类：根据代表每个文档的词项将相似的文档分在一组
  • 导入各种文档格式，如pdf、word
  • 从关系型数据库中导入数据
  • 多语言支持
