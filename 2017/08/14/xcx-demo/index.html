小程序demo

发表于 2017-08-14   |   分类于 小程序

微信小程序（wei xin xiao cheng xu），简称小程序，缩写XCX，英文名wechat little program，是一种不需要下载安装即可使用的应用，它实现了应用“触手可及”的梦想，用户扫一扫或搜一下即可打开应用。

一、首先要有个appID

在微信公众平台官网首页（mp.weixin.qq.com）点击右上角的“立即注册”按钮→选择”小程序”→根据要求填写，详见：https://mp.weixin.qq.com/debug/wxadoc/introduction/


二、基本设置

进入小程序介绍首页，https://mp.weixin.qq.com/wxopen/initprofile?action=home&lang=zh_CN

1、完善“首页”→“小程序发布流程”里的小程序基本信息
2、下载开发工具 下载地址
3、在“用户身份”→开发者→绑定，添加开发者。小程序注册号为管理员，管理员默认有开发者权限
4、在“设置”→“开发设置”里的服务器域名，根据自己的需求配置域名，最基本的得配“request合法域名”；【域名申请、解析自己网上问度娘】

三、开始小demo

1、小程序框架介绍


小程序开发框架的目标是通过尽可能简单、高效的方式让开发者可以在微信中开发具有原生 APP 体验的服务。
框架的核心是一个响应的数据绑定系统。
框架提供了自己的视图层（View）描述语言 WXML 和 WXSS，以及基于 JavaScript 的逻辑层（App Service）框架，并在视图层与逻辑层间提供了数据传输和事件系统，可以让开发者可以方便的聚焦于数据与逻辑上。
上图是框架的最基本的结构，最关键也是必不可少的，是 app.js、app.json、app.wxss 这三个。其中，.js后缀的是脚本文件，.json后缀的文件是配置文件，.wxss后缀的是样式表文件。微信小程序会读取这些文件，并生成小程序实例。

app.js是小程序的脚本代码。我们可以在这个文件中监听并处理小程序的生命周期函数、声明全局变量。调用框架提供的丰富的 API，如本例的同步存储及同步读取本地数据。
app.json 是对整个小程序的全局配置。我们可以在这个文件中配置小程序是由哪些页面组成，配置小程序的窗口背景色，配置导航条样式，配置默认标题。注意该文件不可添加任何注释。
app.wxss 是整个小程序的公共样式表。我们可以在页面组件的 class 属性上直接使用 app.wxss 中声明的样式规则。
微信小程序中的每一个页面的【路径+页面名】都需要写在 app.json 的 pages 中，且 pages 中的第一个页面是小程序的首页。在这里，我们有两个页面，index 页面和 logs 页面，即欢迎页和小程序启动日志的展示页，他们都在 pages 目录下。

每一个小程序页面是由同路径下同名的四个不同后缀文件的组成，如：index.js、index.wxml、index.wxss、index.json。.js后缀的文件是脚本文件，.json后缀的文件是配置文件，.wxss后缀的是样式表文件，.wxml后缀的文件是页面结构文件。
